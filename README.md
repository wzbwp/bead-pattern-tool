# 拼豆图纸解析工具

一个纯前端拼豆图纸解析工具。上传图片后，浏览器会在本地用 Canvas 采样图片，将每个格子转换为拼豆小色块，并生成可预览、可导出的图纸。

## 功能

- 上传 PNG、JPG、WebP 等常见图片格式
- 设置拼豆图纸宽度、高度、采样方式和水平/垂直镜像
- 支持 52 x 52、78 x 78、104 x 104 面板快捷切换
- 上传图片可手动缩放和上下左右移动后重新采样
- 支持默认配色方案和 A-M 全色号/72/48/24/12/8/6 色上限
- 生成带长宽数字和 5 x 5 高亮网格的小色块图纸预览
- 图纸生成后可在画布上手动修改单颗豆子的颜色，并同步刷新用色统计和导出结果
- 自动统一相近轮廓色，并补齐轻微断开的外轮廓
- 统计每个色号需要的颗数
- 导出自动带色号、长宽数字和底部所需颜色清单的 PNG 图纸
- 导出 CSV 用色清单、JSON 原始数据
- 支持 PWA 安装和离线打开应用壳

## 本地打开

这个项目没有依赖，直接打开 `index.html` 即可使用核心功能。

如果需要验证 PWA 安装和离线缓存，需要通过 `localhost` 或 HTTPS 访问。可以在项目目录启动一个本地静态服务：


```bash
npm run dev
```

然后访问：

```txt
http://localhost:5173
```

PWA 实现细节见 [docs/PWA.md](docs/PWA.md)。自定义域名配置见 [docs/CUSTOM_DOMAIN.md](docs/CUSTOM_DOMAIN.md)。后端工程基础整理见 [docs/BACKEND_FOUNDATIONS.md](docs/BACKEND_FOUNDATIONS.md)。

## 部署到 Cloudflare Pages

1. 把本项目推送到 GitHub。
2. 打开 Cloudflare Dashboard，进入 Workers & Pages。
3. 创建 Pages 项目并连接 GitHub 仓库。
4. 构建配置填写：

```txt
Build command: npm run build
Build output directory: .
```

部署成功后会得到类似：

```txt
https://bead-pattern-tool.pages.dev
```

## 部署到 Vercel

1. 把本项目推送到 GitHub。
2. 打开 Vercel，导入仓库。
3. Framework Preset 选择 Other。
4. 构建配置填写：

```txt
Output Directory: .
```

## 颜色配置

当前配色方案为 `默认`，颜色套餐作为自动取色的颜色上限：

- `自动匹配颜色`
- `A-M 全色号 (最多 221)`
- `最多 72 色`
- `最多 48 色`
- `最多 24 色`
- `最多 12 色`
- `最多 8 色`
- `最多 6 色`

默认使用 `自动匹配颜色`：简笔图自动收敛到少量颜色，复杂插画会提高到 24、48 或 72 色，以保留脸部、阴影和小装饰层次。手动选择颜色上限后会严格按该上限输出。

工具会根据选择的颜色上限，从当前图片中自动提取实际需要的代表色，并输出本张图纸的用色列表和数量。色号按 MARD 色卡的色系归属匹配：A 黄、B 绿、C 蓝、D 紫、E 粉、F 红、G 棕、H 黑白灰。白色固定为 `H2`，黑色固定为 `H7`，不再按照颜色生成顺序分配 `A1`、`A2`。

`A-M 全色号 (最多 221)` 会开放完整色号池，但不会强制输出 221 色；如果图片实际只需要约 50 色，就会输出约 50 色。`最多 72 色`、`最多 48 色`、`最多 24 色`、`最多 12 色`、`最多 8 色`、`最多 6 色` 会按对应数量限制代表色。

默认的 `经典还原（原版效果）` 完整使用历史版本的 9 x 9 区域采样和颜色权重，并跳过后续颜色修正。要复现同一张历史图纸，必须使用同一原图、相同宽高和相同颜色上限；例如熊猫示例旧图使用 `80 x 106` 和 `最多 8 色`，需要手动选择 8 色。`平衡还原` 会额外保护连续细线，可在经典模式无法保留细线时手动切换。

上传图片后会先进入图片编辑器，可裁剪、缩放、平移、旋转、水平/垂直翻转，并通过边缘洪水填充一键去除与图片边缘连通的背景。图片编辑器还支持拖拽、画笔、橡皮、取色、填充、直线、矩形、选区和撤销图片编辑，处理后的图片再进入图纸解析流程。`主导色采样（卡通图）` 会优先选择网格内占比明显的有色主体，适合边界清晰的插画；默认仍为 `经典还原（原版效果）`，不会改变历史图纸的默认生成方式。PNG 导出会先显示完整图纸预览，确认后才下载。

上述功能仅使用浏览器 Canvas API，没有新增运行时依赖，现有 GitHub Pages 静态发布流程无需调整。

PWA 能力同样不需要新增依赖：`manifest.webmanifest` 提供安装元数据，根目录 `sw.js` 负责缓存首页、脚本、样式、色卡和图标资源。部署到 HTTPS 后，用户可以把工具安装到桌面或手机主屏；离线时可以继续打开应用并使用已经加载好的前端功能。
